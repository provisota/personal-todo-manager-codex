# Check Migration Skill

After modifying SQLAlchemy models in `backend/app/models/`, verify that a migration already exists for those changes.

## Steps

1. Run `alembic check`:
```bash
cd backend && source .venv/bin/activate && alembic check 2>&1
```

2. Interpret the result:
   - Exit code **0** + `No new upgrade operations detected` — migrations are in sync, all good
   - Exit code **1** + `New upgrade operations detected` — models changed but no migration exists yet

3. If a migration is needed, inform the user and suggest the command:
```bash
cd backend && source .venv/bin/activate && alembic revision --autogenerate -m "describe what changed"
```

Do not create the migration automatically — always ask the user first, since autogenerated migrations require review before being applied.
