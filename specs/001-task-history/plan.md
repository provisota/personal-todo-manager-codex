# Implementation Plan: Task Change History

**Branch**: `001-task-history` | **Date**: 2026-05-12 | **Spec**: `specs/001-task-history/spec.md`  
**Input**: Feature specification from `specs/001-task-history/spec.md`

## Summary

Add per-task change history to the Personal To-Do Manager. Every task save that modifies at least one field automatically creates an immutable history record (two new DB tables). A history button on each task row opens a styled inline panel showing a table of change entries (When / Changed By / Fields Changed). Clicking any row opens a detail modal with a Field / Was / Became table. All UI elements follow the visual design reference captured in `spec.md § Visual Design`.

## Technical Context

**Language/Version**: Python 3.11 (backend) · TypeScript + React 18 (frontend)  
**Primary Dependencies**: FastAPI, SQLAlchemy 2.0 async, Alembic, lucide-react (already present)  
**Storage**: PostgreSQL (prod) · in-memory SQLite (tests)  
**Testing**: pytest + SQLite via `conftest.py` (backend) · Vitest + Testing Library (frontend)  
**Target Platform**: Linux server (backend) · Evergreen browsers (frontend)  
**Project Type**: web-service + SPA  
**Performance Goals**: History list visible within 2 s of button click; record created within 1 s of task save  
**Constraints**: Single GET endpoint — no pagination; all history loaded at once with scroll  
**Scale/Scope**: Personal app — low traffic; history payload is small (≤5 fields × N saves)

## Visual Design Implementation Notes

Visual requirements are defined in `spec.md § Visual Design`. Implementation must match without introducing new CSS libraries.

### History Inline Panel (`TaskHistory.tsx`)

- Outer container: `rounded-lg border border-gray-200 bg-white mx-2 my-1` (card inset from panel edges)
- Table: `w-full border-collapse` with `overflow-hidden rounded-lg`
- Header row: `bg-blue-50` (or equivalent light blue-gray from the app's existing Tailwind palette); `<th>` cells `font-semibold text-sm text-gray-700 px-4 py-2 text-left`
- Data rows: `bg-white hover:bg-gray-50 cursor-pointer`; separated by `border-t border-gray-100`
- `When` column: `whitespace-nowrap` to prevent timestamp wrapping
- `Changed By` column: centered relative to table width
- `Fields Changed` column: flexible remainder; `text-sm text-gray-600`

### Detail Modal (`TaskHistoryModal.tsx`)

- Backdrop: `fixed inset-0 bg-black/50 flex items-center justify-center z-50`
- Modal card: `bg-white rounded-xl shadow-lg p-6 w-full max-w-lg mx-4 relative`
- Close button (X): `absolute top-3 right-3 text-gray-400 hover:text-gray-600`; icon from `lucide-react` (`X` component)
- Table: `w-full border border-gray-200 rounded-lg overflow-hidden`
- Header row: `bg-gray-50` (or `bg-blue-50`); `<th>` cells `font-semibold text-sm px-3 py-2 text-left`
- Data rows: `bg-white`; separated by `border-t border-gray-200`; `<td>` cells `px-3 py-2 text-sm text-gray-700 align-top`
- Long value cells (Was / Became): `max-h-24 overflow-y-auto` to keep modal size predictable

## Key Decisions (from `research.md`)

|---|----------|-----------|
